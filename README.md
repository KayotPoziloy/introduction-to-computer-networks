Знакомство с компьютерными сетями через программу Cisco Packet Tracer


# Теоритические данные:

### IP-адрес
это уникальный числовой идентификатор, который присваивается каждому устройству, подключенному к сети. Он необходим для того, чтобы устройства могли находить друг друга и обмениваться данными.

Структура:
+ В версии IPv4 адрес состоит из четырех десятичных чисел, разделенных точками, например 192.168.1.1 ; Каждое число может принимать значение от 0 до 255.
+ Существует две основных версии IP:
	+ IPv4: 32-битный адрес, что позволяет создать около 4 миллиардов уникальных адресов.
	+ IPv6: 128-битный адрес, был создан для решения проблемы нехватки адресов IPv4, значительно увеличивает кол-во доступных адресов.
+ Могут быть статическими или динамическими.

Функции: 
+ Идентификация: Позволяет различать устройства в сети.
+ Маршрутизация: Упрощает передачу данных между устройствами, указывая, куда конкретно нужно отправлять данные.


### MAC-адрес
(Media Access Control) - это уникальный идентификатор любого устройства, имеющего выход в компьютерную сеть Ethernet. Также известен как аппаратный или физический адрес.

**Зачем нужен:**
+ Используется для идентификации отправителя и получателя данных в сети.
+ В беспроводных сетях MAC-адрес используется для фильтрации, что позволяет разрешить или запретить доступ к сети определенным устройствам.

MAC-адрес идентифицирует устройство в сети.
IP-адрес определяет местоположение устройства.


### Ethernet
это семейство технологий пакетной передачи данных, используемых для компьютерных и промышленных сетей, в основном, для локальных сетей (LAN) и городских сетей (MAN).

Основные принципы:
+ Использует кабельное соединение для передачи данных и MAC-адрес для идентификации отправителя и получателя данных.
+ **Функционирование:** Устройство, желающее отправить данные, создает кадр данных, который отправляется на сетевой адаптер, преобразующий его в электрический или оптический сигнал для передачи по кабелю. Коммутатор или Маршрутизатор получает сигнал, проверяет MAC-адрес получателя и направляет кадр данных к соответствующему устройству.

### Протокол ICMP
(Internet Control Message Protocol) - это сетевой протокол, используемый для диагностики, передачи сообщений об ошибках и предоставления информации об управлении и состоянии в [[IP-адрес|IP]]-сетях. 

Принцип работы ICMP:
+ Основан на протоколе IP. Сообщение об ошибке или управляющая информация инкапсулируется в IP-пакет с новым заголовком IP для отправки обратно отправителю.
+ ICMP является ненадежным протоколом, так как не контролирует доставку и ее правильность.
Особенности ICMP:
+ Если ICMP-пакет был потерян при передаче, новый не генерируется.
+ ICMP-сообщения не создается в ответ на широковещательные или групповые запросы для предотвращения перегрузки сети.
+ Если IP-пакет был поврежден при передаче, ICMP-сообщение отправляется источнику.
Применение ICMP:
+ Диагностика сети: часто используется в утилитах [[ping]] и [[tracetoute]] для проверки связи между узлами.
+ Сообщения об ошибках: ICMP сообщает об ошибках, возникающих при передаче данных, например, о недоступности узла, превышении времени жизни пакета.
+ Управление трафиком: Маршрутизаторы могут отправлять ICMP-сообщения, чтобы указать более выгодный маршрут к узлу назначения.
Безопасность:
+ ICMP может быть использован для сетевых атак, таких как ICMP flood или ping of death.
+ Для повышения безопасности сети многие применяют фильтрацию ICMP.

### Протокол ARP
Для определения [[MAC-адрес|MAC-адреса]] по известному [[IP-адрес|IP-адресу]] используется протокол разрешения адреса (Address Resolution Protocol).

Основные функции ARP:
+ Преобразование адресов: ARP позволяет устройствам в локальной сети находить MAC-адреса, необходимые для передачи данных на уровне [[Ethernet]]. 
	*Если компьютер А хочет отправить данные компьютеру В, он должен знать MAC-адрес B, даже если ему известен IP-адрес*.
+ Широковещательные запросы: Когда устройство не может найти нужный MAC-адрес в своей ARP-таблице (кэше), оно отправляет широковещательный запрос всем устройствам в сети с просьбой сообщить свой MAC-адрес, если у них совпадает IP-адрес.
+ ARP-таблица: Для повышения эффективности работы, ARP хранит информацию о соответствиях IP и MAC-адресов в таблице, что позволяет избегать повторных запросов и экономить сетевые ресурсы.

Принцип работы ARP:
1. Устройство формирует ARP-запрос с указанием IP-адреса получателя и рассылает его всем устройствам в локальной сети.
2. Все устройства проверяют, соответствует ли указанный IP их собственному. Если совпадение найдено, устройство отправляет ответ с указанием своего MAC-адреса.
3. Отправитель получает ответ и обновляет свою ARP-таблицу для будущих обращений.

### ping
`ping` отправляет ICMP echo-запросы и получает ICMP echo-ответы, чтобы определить, доступен ли хост и измерить задержку


### tracetoute
показывает маршрут между хостами и время обработки пакета на каждом участке маршрута.


### Маршрутизатор
Это сетевое устройство, предназначенное для соединения различных сегментов сети и передачи данных между ними

Основные функции маршрутизатора:
1. Маршрутизация: определение оптимального пути для передачи пакетов данных на основе таблиц маршрутизации. Маршрутизатор использует [[IP-адрес|IP-адреса]] в заголовках пакетов и решает, куда их направить.
2. Связь между локальной и глобальной сетью
3. Управление скоростью передачи данных для предотвращения перегрузок сети


### Коммутатор
Устройство, которое используется для соединения нескольких устройств в локальной сети и передачи данных между ними.

Основные функции:
+ Передача данных только на порт, к которому подключен получатель, на основе [[MAC-адрес|MAC-адресов]]. Это уменьшает трафик в сети.
+ Автоматически создает [[Протокол ARP#^06a1d2|таблицу MAC-адресов]].


### SMTP
Simple Mail Transfer Protocol - протокол, используемый для отправки электронной почты. Обеспечивает передачу сообщений между серверами и клиентами.

Основные аспекты:
+ Используется исключительно для отправки писем. Для получения используются другие протоколы, например, [[POP3]]
+ Механизм работы:
	+ Клиент устанавливает соединение с сервером через [[TCP]]
	+ Происходит рукопожатие, где клиент идентифицирует себя командой HELO/EHLO
	+ После успешной передачи соединение завершается командой QUIT
+ Поддерживает только текстовые сообщения, для отправки вложений используется расширение MIME.


### POP3
Post Office Protocol 3 - это протокол, предназначенный для получения писем с почтового сервера. 

Основные характеристики: 
+ Позволяет клиенту загружать письма с сервера на локальное устройство.
+ Механизм работы:
	+ Клиент подключается к серверу через [[TCP]]
	+ Пользователь проходит аутентификацию с помощью логина и пароля.
	+ Клиент может выполнять команды, такие как LIST (список писем), RETER (загрузка письма) и DELE (удаление письма)
	+ После завершения загрузки соединение закрывается

POP3 подходит только для пользователей, которые работают с почтой на одном устройстве, так-как этот протокол не поддерживает синхронизацию между устройствами, эти функции доступны в IMAP.


### DNS
Domain Name System - это распределенная система, которая преобразует удобные для человека доменные имена (www.example.com) в [[IP-адрес|IP-адреса]], используемые компьютерами для связи в сети. DNS - как телефонная книга в интернете.

### Вопросы к лр 3
1. Сколько IP-адресов и MAC-адресов у маршрутизатора?
	У маршрутизатора кол-во IP- и MAC-адресов зависит от числа его интерфейсов (портов). Сколько интерфейсов, столько и адресов.

2. Что такое шлюз? Для чего он используется?
	Это точка в сети, которая служит выходом из одной сети в другую, обеспечивая связь между разными сетями.
	- Обеспечивает передачу данных между сетями с разными протоколами или технологиями.
	- Марщрутизация трафика между сетями
	- Преобразование протоколов
	- Обеспечение безопасности (фильтрует трафик)
	- Мониторинг и диагностика

3. Возможно ли подключиться к сети с другим идентификатором без настройки шлюза?
   	Если на устройстве не настроен шлюз, оно может обмениваться только устройствами в своей подсети. Без шлюза, устройство не знаект куда направлять пакеты вне своей подсети

4. Что такое запрос ARP и ответ на него? Для чего они нужны?
	ARP запрос нужен для сопоставления IP- и MAC-адресов устройств. Без этой информации, устройства не смогут обмениваться информацией. 

5. Что такое таблица ARP и что такое таблица MAC? Чем они отличаются?
	- ARP-таблица содержит пары IP-MAC для устройств.
	- MAC-таблица хранится в коммутаторах и содержит соответствия MAC-адрес устройства - порт коммутатора.

7. Как работает маршрутизатор при перенаправлении пакета во внешнюю подсеть?
	Когда маршрутизатор получает пакет, он сначала проверяет IP-адрес назначения и определяет, он принадлежит локальной подсети или внешней сети.
	Если находится во внешней сети, марщрутизатор обращается к своей таблице маршрутизации, чтобы найти следующий маршрутизатор, через который нужно отправить пакет.
	Если в таблице маршрутизации есть конкретный маршрут к подсети назначения, пакет отправляется на соответствующий интерфейс, в противном случае используется маршрут по умолчанию, который ведет к провайдеру.
	При выходе пакета из подсети маршрутизатор заменяет частный IP-адрес на публичный.

8. Если мы отправим запрос во внутреннюю сеть, какая запись появится в таблице ARP устройства? А если запрос будет направлен во внешнюю сеть?
	При внутреннем запросе сохраняется IP-MAC адресата.
	При внешнем запросе сохраняется IP-MAC маршрутизатора, через который происходит переправка во вне.

9. На каких уровнях модели OSI работают сетевые устройства?
	- Физический уровень (1-й уровень) - здесь работают концентраторы (хабы), повторители. Они передают электрические или оптические сигналы без анализа содержимого.

	- Канальный уровень (2-й уровень) - работают коммутаторы, мосты, сетевые карты. Они обрабатывают кадры, управляют MAC-адресацией и обеспечивают надежную передачу данных в пределах локальной сети.

	- Сетевой уровень (3-й уровень) - здесь функционируют маршрутизаторы (роутеры), которые занимаются маршрутизацией пакетов по IP-адресам между разными сетями.

10. Чем сетевые устройства отличаются от конечных?
	- Конечные устройства - это источники и получатели данных.
	- Сетевые устройства - это посредники, которые обеспечивают передачу, маршрутизацию и управление трафиком между конечными устройствами.

11. Что такое широковещательный запрос? Запрос по какому протоколу является широковещательным?
	Широковещательный запрос - это запрос, который отправляется одновременно всем устройствам в локальной сети, а не конкретному узлу.
	Такой запрос использует ARP

12. Какие данные должен хранить коммутатор, чтобы выполнять переключение устройств?
	Должен хранить соответствия MAC-адрес устройства и соответствующий порт комматутатора

13. Каков параметр TTL в команде ping? Что это значит?
	- TTL - время жизни пакета, ограничивающее число маршрутизаторов на пути (используется для предотвращения бесконечного циркулирования пакетов).
	- В пинг задается параметров -i TTL (максимальное значение - 255)
	
14. Почему первый ping-пакет не приходит при первой попытке отправить запрос во внешнюю сеть?
	Отсутствие записи в ARP-таблице шлюза. При первом обращении сначала нужно узнать MAC-адрес шлюза через ARP-запрос.
